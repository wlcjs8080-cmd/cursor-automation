# Python 일반 규칙

- import는 파일 최상단에 모아두라 (표준 라이브러리 → 서드파티 순)

# xlwings 사용 규칙

- xlwings만 사용한다 (openpyxl 절대 금지)
- 엑셀 파일을 열 때 반드시 try-finally로 감싸서 Excel 프로세스를 종료하라
- 마스터 시트 열 때 필터 자동 해제 후 검색하라
- xw.books.active는 Step 2처럼 사람이 이미 열어둔 파일에 접근할 때만 사용하라
- 새 파일을 열거나 생성할 때는 xw.Book() 사용


# 엑셀 시트 접근 규칙

- 시트명은 정확히 입력하라 (오타 주의)
  - 설비 Master Sheet (O)
  - New Alarm 및 사용Part 이력 (O)
  - Alarm 및 사용Part 이력 (X ← 이건 틀린 이름)
- 머릿말 행과 데이터 시작 행을 반드시 구분하라
  - 설비 Master Sheet: 머릿말 6행, 데이터 7행부터
  - New Alarm 및 사용Part 이력: 머릿말 2행, 데이터 3행부터
  - 스케줄 시트: HEADER_ROWS 기준, 데이터는 머릿말+1행부터
- 열 번호는 반드시 실제 확인된 값을 사용하라 (엑셀 화면과 다를 수 있음)


# 파일 처리 규칙

- 실행 전 자동 백업 생성 (파일명_backup_YYYYMMDD_HHMMSS.xlsx)
- backup 포함 파일은 처리 대상에서 제외
- ~$ 임시 파일 자동 제외
- 파일명에 ~$가 붙은 파일은 엑셀이 만든 임시 파일이므로 절대 열지 마라


# 에러 처리 패턴

- S/N 마스터 미발견 → 오류 메시지 출력 후 해당 건 건너뜀 (전체 중단 금지)
- 방문일정 없음 → 오류 메시지 후 건너뜀
- 고객사명 누락 → 오류 메시지 후 건너뜀
- 필수값 누락 → 해당 항목 누락 메시지
- REPORT 폴더 없음 → 오류 메시지 후 건너뜀
- 한 건의 에러가 전체 실행을 중단시키면 안 됨


# Step 1 코드 규칙

- 스케줄 시트에서 AB열=O + AC열=비어있는 행 찾기
- 3월 블록(HEADER_ROWS 183)부터만 실행
- 날짜 우선순위: N열(3차) > M열(2차) > L열(1차)
- 레포트 생성 후 AC열에 "완료" 표시
- SET UP 조회: 설비 Master → New Alarm 순으로 fallback


# Step 2 코드 규칙

- K57~K63(Part별 이전 교체일)만 처리
- 이전 점검일(V11)은 Step 1에서 처리 완료됨 → Step 2에서 제외
- 열려있는 레포트에서 직접 읽기/쓰기 방식 (xw.books.active 사용)
- N11에서 S/N 읽기, I57~I63에서 품번 읽기
- New Alarm 및 사용Part 이력에서 S/N(L열) + 품번(AC열) 매칭
- 오늘 이전 가장 최근 작업일자(P열) → K57~K63에 입력
- I57~I63에 품번이 비어있으면 해당 행은 건너뜀
- 사람이 화면에서 결과 확인 후 직접 저장
- 파트명: B57~B63(좌측), O57~O61(우측) — 품번만 매칭 (S/N 무관), 마스터 AB열(col28)
- 규격: F57~F63(좌측), S57~S61(우측) — 품번만 매칭 (S/N 무관), 마스터 AK열(col37)
- 사용일: L57~L63(좌측), Y57~Y61(우측) — 점검일(V8) - 이전교체일, 교체이력 없으면 0
- 우측 품번: V57~V61(col22), 우측 이전교체일: X57~X61(col24)
- 매칭 없으면 파트명, 규격, 이전교체일에 "교체이력 없음" 출력
- 글자 크기: 좌측은 I열, 우측은 V열 기준으로 동일 적용
- ShrinkToFit = True 적용 (단, 병합셀에서는 미작동)
- 품번 셀(I57~I63, V57~V61)에도 ShrinkToFit = True 적용
- 행 높이 AutoFit: 값 입력 완료 후 57~63행에 적용
