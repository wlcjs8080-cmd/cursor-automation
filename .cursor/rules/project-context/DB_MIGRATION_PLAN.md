# DB 마이그레이션 계획 (2026-03-05 확정)

## 목표
마스터 시트와 스케줄 시트의 동시 접근 충돌을 해결하기 위해 SQLite DB를 도입한다.
4명이 동시에 자동화를 실행해도 기다림 없이, 충돌 없이 작동하는 구조를 만든다.

## 현재 문제점
- xlwings로 엑셀 파일을 열면 파일 잠금(lock)이 걸려 다른 사람이 동시에 사용 불가
- 마스터 시트 1개를 여러 명이 동시에 읽기/쓰기 불가능
- 스케줄 시트도 자동화 실행 시 엑셀을 열어야 하므로 동시 실행 불가

## 해결 구조: SQLite DB 도입

### 파일 구성
- master.db: 마스터 시트 데이터 (영구 보관, 계속 누적)
- 스케줄 DB: 스케줄 시트 데이터 (자동화 시작 시 최신 내용 반영, 덮어쓰기 아닌 추가/업데이트 방식)
- 레포트 파일: 기존과 동일하게 엑셀 파일로 유지 (각자 독립적)

### 전체 업무 흐름
1. 사람이 SharePoint 스케줄 시트에 입력 (기존과 동일)
2. 자동화 시작 버튼 → 스케줄 엑셀을 읽어서 스케줄 DB에 추가/업데이트
3. Step 1: 스케줄 DB에서 미완료 행을 읽고, 레포트 템플릿 복사하여 생성, 스케줄 DB에 "완료" 표시
4. Step 2: 각자 레포트 열고 품번 입력 후 실행 → 마스터 DB에서 이전 교체일 조회 (읽기만)
5. Step 3: 레포트 + 스케줄 DB 데이터를 읽어서 마스터 DB에 새 행 기록 (쓰기)
6. 퇴근 시: 내보내기 버튼 → 마스터 DB 전체를 마스터 엑셀로 저장, 스케줄 DB "완료"를 원본 스케줄 엑셀에 반영

### SQLite 동시 접근 특성
- 동시 읽기: 제한 없음 (몇 명이든 동시에 읽기 가능)
- 동시 쓰기: SQLite가 자동으로 순서 관리 (0.1초 대기 후 순차 처리, 데이터 유실 없음)
- 파일 잠금: 엑셀처럼 전체 파일을 잠그지 않음

### DB 초기 설정 (최초 1회)
- 기존 마스터 엑셀의 전체 데이터를 master.db로 변환하는 스크립트 실행
- 이후 Step 3에서 새 데이터가 자동 누적됨

### 코드 변경 범위
- 변경됨: 마스터 시트를 읽고 쓰는 함수들 (xlwings → SQLite)
- 변경됨: 스케줄 시트를 읽는 함수들 (xlwings → SQLite, Step 1 시작 시 DB 생성)
- 변경 없음: 레포트 템플릿 복사 (shutil.copy2 그대로)
- 변경 없음: 레포트 셀 채우기 (xlwings 그대로)
- 변경 없음: 레포트 읽기 (xlwings 그대로, Step 2/3에서 각자 레포트 열기)
- 추가: DB 초기화 스크립트 (마스터 엑셀 → DB 변환)
- 추가: 내보내기 스크립트 (DB → 마스터 엑셀, DB → 스케줄 엑셀 완료 반영)

### 스케줄 DB 관리 규칙
- 덮어쓰기 금지: 새로 자동화를 시작해도 기존 "완료" 표시를 유지
- 추가/업데이트 방식: 새 행은 추가, 기존 행은 상태만 업데이트
- 이전 사람이 처리한 "완료"는 보호됨

### 마스터 DB 관리 규칙
- 데이터 영구 보관 (삭제 없음, 계속 누적)
- 내보내기는 DB의 복사본을 만드는 것이며 DB를 비우지 않음
- 다음날에도 전날 데이터가 그대로 유지됨

### 안정성 검증 결과 (시나리오 테스트 완료)
- 4명이 서로 다른 시간에 자동화를 실행해도 충돌 없음 확인
- 동시에 마스터 DB에 쓰기를 해도 SQLite가 순차 처리하여 안전
- 한 명이 완료하기 전에 다른 사람이 시작해도 각자 DB를 통해 독립적으로 작동
- 유일한 규칙: 퇴근 시 한 명이 내보내기 실행

### 구현 우선순위
1. Step 1~3 통합 테스트 완료 (현재 진행 중)
2. DB 초기화 스크립트 작성 (마스터 엑셀 → master.db)
3. Step 1 코드 수정 (스케줄 DB 생성 + DB에서 읽기)
4. Step 2 코드 수정 (마스터 DB에서 읽기)
5. Step 3 코드 수정 (스케줄 DB 읽기 + 마스터 DB 쓰기)
6. 내보내기 스크립트 작성 (DB → 엑셀)
7. 전체 통합 테스트 (4명 동시 실행 시나리오)
8. .exe 앱 변환 (GUI에 자동화 버튼 + 내보내기 버튼)

### 배경: 비교 검토 완료
- 클라우드 전면 전환 (Office Scripts + Power Automate): TypeScript 전면 재작성 필요, 3~4주, 비추천
- 마스터 5개 분리 방식: 안정적이나 합치기 복잡, 용량 5배, 관리 부담
- SQLite DB 방식: 기존 코드 최소 수정, 동시 접근 안전, 기다림 제로 → 채택
