# 프로젝트 개요

## 프로젝트명
엑셀 업무 자동화 시스템 (스케줄 → 레포트 → 마스터 연동)

## 한 줄 요약
스케줄 시트에 업무를 기록하면, 자동으로 레포트 파일을 만들고 마스터 시트에서 필요한 정보를 가져와 채워주는 프로그램

## 핵심 목표
- 비개발자(DK) 포함 5명이 서버에서 사용할 수 있는 .exe 앱
- xlwings 기반 엑셀 자동화
- 수동 반복 작업 제거

# 시트 구조 (실제 확인 완료)

## 스케줄 시트 (Sheet1)
- HEADER_ROWS = [93, 138, 183, 229, 274, 319, 364] (93=1월 ~ 364=7월)
- 1월(93), 2월(138)은 수동 처리 완료 → 실행 제외, 183(3월)부터 실행
- 각 블록 데이터 범위: 머릿말+1행 ~ 머릿말+41행
- 열 매핑: J=접수일자, K=방문계획, L=1차방문, M=2차방문, N=3차방문, O=미준수사유, P=고객사, Q=담당자, R=접수자, S=MAT작업자, T=업무내용, U=유무상, V=인원, W=공정, X=라인, Y=MODEL, Z=설비호기, AA=S/N, AB=방문완료, AC=처리완료
- 트리거: AB열 = O + AC열 = 비어있음
- 날짜 우선순위: N열(3차) > M열(2차) > L열(1차)

## 설비 Master Sheet
- S/N = J열(col10)
- SET UP = M열(col13)
- 머릿말 = 6~7행(병합, 셀주소 6행), 데이터 = 8행부터

- S/N = L열(col12)

- 머릿말 = 2행, 데이터 = 3행부터

## 레포트 템플릿 (PM REPORT 새양식)
- E10 ← P열(고객사)

- I57~I63 = 품번 (사람이 수동 입력)
- K57~K63 = Part별 이전 교체일 (Step 2에서 자동 입력)

- F57~F63 = 규격 (Step 2에서 자동 입력, 좌측)
- L57~L63 = 사용일 (Step 2에서 자동 입력, 좌측)
- O57~O61 = 파트명 (Step 2에서 자동 입력, 우측)
- S57~S61 = 규격 (Step 2에서 자동 입력, 우측)

# 영구 참고사항

## AC열 표시 규칙
- 기존완료 → Step 0 (수동 처리 건)
- 완료 → Step 1 (자동화 생성 건)
- AC열에 값이 있으면 처리 대상에서 제외

## 오류 처리
- S/N 마스터 미발견 → 오류 메시지 후 건너뜀
- 방문일정 없음 → 오류 메시지 후 건너뜀
- 고객사명 누락 → 오류 메시지 후 건너뜀
- 필수값 누락 → 해당 항목 누락 메시지
- REPORT 폴더 없음 → 오류 메시지 후 건너뜀

## 속도 개선 (완료)
- 현재 마스터 시트를 한 행씩 개별 읽기 방식 → 느림

- Step 1, Step 2에 적용 완료
